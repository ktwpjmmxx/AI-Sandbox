# Elyza-7B v7

> **471件の厳選データセット×最大2.54エポック学習。Lossは0.5736まで劇的に改善。しかし、2.5エポック付近で「壊れたレコード現象（ループ）」と「ハルシネーション」が発現。この失敗により、最適解が1.5〜2.0エポックの間に潜んでいることを突き止めた、極めて実りある実験。**

#### Loss・Norm推移の詳細分析
- **安定期 (〜1.5 Epoch / checkpoint-45推定)**: Lossが0.6台で安定。日本語の整合性と専門性のバランスが最も取れていたと推測される。
- **不安定期 (2.0〜2.4 Epoch)**: `grad_norm` が **0.48** まで上昇。モデルが既存の知識（ベースモデルの良識）を強引に書き換え始めた予兆。
- **崩壊期 (2.5 Epoch〜 / checkpoint-60)**: Lossは最小値を更新したが、推論時に同一フレーズを繰り返す「ループ現象」が発生。

## 推論テストの分析

### 1.0 Epoch時点 (checkpoint-30)
- **判定**: **「喜（安定版）」**
- **評価**: 日本語は極めて自然。システムプロンプトに従い「できないこと」を拒絶する自制心が機能している。
- **課題**: 専門知識の「キレ」がベースモデルに近い。まだ「IT法務の檻」への染まり方が浅い。

### 2.54 Epoch時点 (checkpoint-60)
- **判定**: **「憂（過学習・崩壊）」**
- **評価**: 専門用語を多用するが、同じ文章を繰り返す。実在しない法律「デジタル・ネイティブ・エミュレーション法」を創作。
- **結論**: モデルが「正気」を保てる限界点（Overfit）を明確に突破した。

## v7で得た決定的な教訓（v8への戦略）

### 1. 最適解は「1.5〜2.0 Epoch」の間にある
1.0では「知識不足」、2.5では「崩壊」。今回の実験により、ベースモデルの良識を保ちつつ専門性を最大化できる**「黄金のスイートスポット」**がその中間に存在することが確定した。

### 2. チェックポイント配置の戦略的ミス
今回は `save_steps` が大きすぎたため、最も美味しいはずの「1.5エポック付近」を物理的に抽出できなかった。**「モデルの抽出失敗」こそが最大の収穫であり、次回の成功の鍵**となる。

### 3. Grad Normの監視が「避難訓練」になる
`grad_norm` が 0.4 を超えたタイミングが、モデルが「嘘」を突き始めるターニングポイント。次回はこれを目安に早期停止（Early Stopping）の判断を行う。

## v8への改善ロードマップ

1.  **チェックポイントの超高頻度化**
    - `save_steps = 10` （約0.2エポックごと）に設定し、1.0〜2.0エポックの間を「1.2 / 1.4 / 1.6 / 1.8」と細かくスライスして保存する。
2.  **学習データの「拒絶」表現の強化**
    - 1.0エポックで拒絶が甘かったのは、ベースモデルの「親切心」が勝っているため。より毅然とした、定型的な拒絶パターンを強調する。
3.  **短距離決戦へのシフト**
    - 10エポック回す必要はなく、最大3エポック以内での「最高の一瞬」を拾い上げる戦略に切り替える。
