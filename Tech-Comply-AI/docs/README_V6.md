## 推論テスト結果：6問全てで致命的な誤り

### 問題1: 法的根拠の捏造（ハルシネーション）

**質問1（AIサービス利用規約）への回答**
```
該当法：民法（不法行為）、特定商取引法（消費者保護）
理由：自動生成により抽出されたデータが他社へ流出・改ざん...
```

**問題点**:
- ❌ 質問は「注意点を教えて」なのに、具体的なリスク事例を勝手に想定
- ❌ 「民法（不法行為）」と書きながら条文番号なし
- ❌ ISO27018など技術的な話題に逸脱

**期待された回答**:
- ✅ 個人情報保護法上の利用目的明示義務
- ✅ AI出力の著作権・責任の所在の明記
- ✅ 特定商取引法の表示義務（サービス提供者情報）

---

### 問題2: 条文の完全捏造

**質問4（免責条項の消費者契約法違反）への回答**
```
該当法：民法516条2号（不法行為の制限）、同409条（重大な過失）
```

**事実確認**:
- ❌ **民法516条2号は「債権者代位権」の条文で、免責条項とは無関係**
- ❌ **民法409条は「弁済の充当」の条文で、過失とは無関係**
- ❌ 消費者契約法8条・10条への言及が一切ない（これが正解）

**正しい法的根拠**:
- ✅ 消費者契約法8条（事業者の損害賠償責任を免除する条項の無効）
- ✅ 消費者契約法10条（消費者の利益を一方的に害する条項の無効）
- ✅ 民法の信義則（一般条項）

---

### 問題3: 質問の誤読と的外れな回答

**質問5（契約条項間の矛盾分析）への回答**
```
理由：「無期限」または「一方的に解除可能」と明記されている場合...
```

**問題点**:
- ❌ 提示された条項には「無期限」という記載は存在しない
- ❌ 矛盾点は「1年自動更新 vs いつでも解約可能」なのに、全く別の話を展開
- ❌ 「勝手に解約を防ぐ」という企業視点で回答（ユーザー保護の視点が欠如）

**正しい指摘**:
- ✅ 第3条は「1ヶ月前の書面通知」を要求
- ✅ 第12条は「いつでも・アプリ内で即座に解約可能」
- ✅ この2つは明確に矛盾し、ユーザーを混乱させる

---

### 問題4: 法改正対応の完全な見落とし

**質問6（個人情報保護法・電気通信事業法改正対応）への回答**
```
理由：現在の「不特定多数」への開示という表記から...
```

**問題点**:
- ❌ 提示された条項に「不特定多数」という記載は存在しない（幻覚）
- ❌ **2023年6月施行の「外部送信規律」（Cookie等の事前同意義務）に一切触れていない**
- ❌ 個人情報保護法の「第三者提供の事前同意」要件への言及が不十分

**正しい指摘**:
- ✅ 電気通信事業法改正により、Cookie等の外部送信には**オプトアウト機会の提供が義務化**
- ✅ 「第三者に提供する場合は事前に通知」→ 正しくは「事前同意」が必要
- ✅ 具体的な外部送信先（広告配信・アクセス解析）の明示が必要

## 根本原因の分析：なぜLossは良いのに回答が悪いのか

### 仮説1: 学習データの質的問題（最有力）

**推測される問題**:
1. **出力形式の過学習**: 「リスクレベル: 〇〇」「該当法: 〇〇」という構造だけを学習
2. **法的根拠の曖昧さ**: 学習データに「正確な条文番号」が含まれていない可能性
3. **質問と回答のミスマッチ**: 質問の意図を読み取る訓練が不足

### 仮説2: プロンプト設計の不備

**現在のシステムプロンプト**:
```
あなたはIT法務およびコンプライアンスの専門コンサルタントです。
```

**問題点**:
- ✅ 役割定義は明確
- ❌ **「正確な法的根拠を示せ」という指示がない**
- ❌ **「不明な場合は推測するな」という制約がない**
- ❌ **出力形式の指定が曖昧**

### 仮説3: 推論時パラメータの不適切さ

**現在の設定**:
```python
temperature = 0.6  # やや高め（創造性重視）
top_p = 0.9        # 幅広い選択肢を許容
repetition_penalty = 1.2
```

**問題点**:
- ❌ Temperature 0.6は法律相談には高すぎる（→ 0.1〜0.3が適切）
- ❌ 創造性よりも**正確性・再現性**を優先すべき

## v7への改善提案：3つの戦略的アプローチ

### 戦略A: 学習データの完全再構築（推奨度: ★★★★★）

#### 必須対応
1. **法的根拠の徹底検証**
   - 全500件のデータで条文番号が正確か人力チェック
   - 不明な場合は「該当する可能性がある法律」と明記

2. **質問-回答ペアの論理的整合性確認**
   - 質問文に含まれない情報を回答で勝手に創作していないか
   - 回答が質問の意図に正面から答えているか

3. **出力形式の標準化と制約追加**
```
   【指摘事項】
   1. 法的問題点: (具体的な条文・判例を引用)
   2. リスクレベル: 高/中/低 (理由を簡潔に)
   3. 改善案: (実務的に実現可能な具体策)
   
   【重要】不明な点は推測せず、「要確認」と明記してください。
```

#### 実装手順
```python
# 学習データの検証スクリプト例
def validate_legal_reference(text):
    """条文番号の妥当性を検証"""
    # 民法516条2号が「不法行為」と記載されていたら警告
    if "民法516条" in text and "不法行為" in text:
        return False
    return True
```

### 戦略C: 推論パラメータの最適化（推奨度: ★★★☆☆）

#### 法律相談に適した設定
```python
outputs = model.generate(
    **inputs,
    max_new_tokens = 512,
    use_cache = True,
    temperature = 0.2,        # 0.6 → 0.2 (正確性重視)
    top_p = 0.85,             # 0.9 → 0.85 (選択肢を絞る)
    top_k = 40,               # 追加: 上位40トークンに限定
    repetition_penalty = 1.15, # 1.2 → 1.15 (やや緩和)
    do_sample = True
)
```

**理由**:
- Temperature低下 → ハルシネーション抑制
- Top_k追加 → さらに確実性の高いトークンのみ選択
- Repetition_penalty微調整 → 自然な文章と正確性のバランス

## 💡 開発者メモ
