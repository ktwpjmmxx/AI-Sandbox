## v2 → v3 変更点

### 1. データの改善
**v2:**
```json
{
  "output": "{\n  \"リスクレベル\": \"高\",\n  ...\n} この広告は...（余計な説明が続く）"
}
```

**v3:**
```json
{
  "output": "{\"リスクレベル\":\"高\",\"該当法\":\"景品表示法\",\"理由\":\"...\",\"修正案\":\"...\"}"
}
```
- JSON以外のテキストを完全削除
- 改行を削除してコンパクト化

### 2. プロンプトの強化

**v2:**
```
回答は必ず指定されたJSON形式で、それ以外の文章は含めないでください。
```

**v3:**
```
回答は必ずJSON形式のみで出力し、JSON以外の文章は一切含めないでください。
JSONの開始{から終了}まで以外は何も出力しないでください。
```
- より厳格な指示
- 「のみ」「一切」などの強調

### 3. パラメータ調整

| パラメータ | v2 | v3 | 効果 |
|-----------|----|----|------|
| `num_train_epochs` | 5 | **7** | より深く学習 |
| `learning_rate` | 1e-4 | **8e-5** | より安定 |
| `temperature` | 0.7 | **0.3** | ランダム性↓ |
| `repetition_penalty` | 1.1 | **1.15** | 繰り返し抑制強化 |
| `no_repeat_ngram_size` | なし | **3** | 新規追加 |

### 4. 推論の改善

**新機能:**
- JSON抽出機能（正規表現で`{...}`のみ抽出）
- 繰り返し抑制の強化
- n-gram繰り返しの禁止

## トラブルシューティング

### 問題1: まだJSON後にテキストが出る

**原因**: 学習不足

**対処法**:
1. エポック数を10に増やす
2. データの確認（JSON形式のみか）
3. さらに学習を継続

### 問題2: JSON形式成功率が50-70%

**原因**: 推論パラメータが緩い

**対処法**:
```python
# inference_v3.py を編集
temperature=0.2,  # 0.3 → 0.2
top_p=0.85,       # 0.9 → 0.85
```

### 問題3: 学習中にメモリ不足

**対処法**:
```yaml
# config_v3.yaml を編集
training_arguments:
  gradient_accumulation_steps: 16  # 8 → 16
```

- **エポック**: 7
- **バッチサイズ**: 実質8 (1 × 8 gradient accumulation)
- **学習率**: 8e-5
- **スケジューラ**: cosine

### 推論
- **temperature**: 0.3
- **top_p**: 0.9
- **repetition_penalty**: 1.15
- **no_repeat_ngram_size**: 3
