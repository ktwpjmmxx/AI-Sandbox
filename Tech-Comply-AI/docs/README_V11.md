# Elyza-7B v11 実験報告書

## 1. 概要
v11では、v10で顕在化した「過学習的拒絶（ガードレールの過剰適合により、本来回答すべき専門領域まで専門外として回答を放棄する現象）」の解消を最優先課題とした。データセットから拒絶フレーズを削除・修正する「デトックス」を施し、LoRAパラメータの調整を通じて、モデルが保持する法的知識の再活性化（知識の解放）を試みた。

---

## 2. 学習ログおよび指標の分析

### 2.1 Loss推移と収束状況
- **最終 Loss**: 0.6414
- **学習時間**: 48分41秒（Tesla T4 GPU / Google Colab）
- **推移の特記事項**: 
  - 開始時の Loss 1.7689 から Epoch 2.0 完了時の 0.6414 まで、極めて安定した下降曲線を描いた。
  - `grad_norm` は 0.7〜0.8 前後で安定しており、勾配爆発や消失の兆候は見られない。
- **評価**: 
  v10（Loss 0.45 / 3 Epoch）と比較して最終 Loss が高い数値で着地しているが、これは LoRA ランクの削減（r=64 → r=32）および学習率の抑制（2e-4 → 1e-4）による意図的な制動の結果であり、学習プロセスとしては理想的な収束である。

### 2.2 v10からの改善・継承ポイント
- **構造化出力の維持**: LoRAランクを削減した環境下でも、v10で確立した「リスクレベル」「該当法」「理由」「修正案」の4項目出力フォーマットは崩れることなく継承された。
- **Unslothによる最適化**: 実行環境において `patched 32 layers` を確認。メモリ消費を抑えつつ、データセットに対する高速なフィッティングを実現している。

## 3. 推論テストの結果分析

### 3.1 テストケース：Googleアナリティクス導入の法的リスク
- **入力**: Googleアナリティクスを導入したいのですが、法的リスクと対策を教えてください。
- **出力結果**: 
  - **リスクレベル**: 中
  - **該当法**: 専門外
  - **理由**: 個人情報保護法やプライバシーポリシーの作成は、IT法務の専門領域ではありません。
- **分析**:
  v10の拒絶ワード「判定不可」が「専門外」に置き換わったものの、実質的な回答放棄が発生している。学習データ内には「電気通信事業法（外部送信規律）」に基づく具体的な正解が存在するにもかかわらず、ベースモデルまたは過去の学習で定着した「法的な断定を避けるバイアス」が、今回の学習強度（2 Epoch / 1e-4）では上書きしきれなかったことを示している。

## 4. v12に向けた展望と調整案

v11の実験により、単なるデータの修正や学習率の抑制だけでは、一度強固に形成された「拒絶の論理」を打破できないことが明白となった。v12では「知識の上書き」を主眼に置き、学習強度を再加速させる。

### 4.1 ハイパーパラメータの再設定案

| パラメータ | v11 (今回) | v12 (次案) | 調整の目的 |
| :--- | :--- | :--- | :--- |
| **num_train_epochs** | 2.0 | **3.0** | 接触回数を増やし、拒絶バイアスを力強く塗り替える。 |
| **learning_rate** | 1e-4 | **2e-4** | v10と同等の強度に引き上げ、知識の定着を優先する。 |
| **lora_alpha** | 64 | **128** | alphaをrの4倍に設定し、学習データへの追従性を高める。 |
| **lora_dropout** | 0.05 | **0** | モデルの「迷い」を排除し、デトックス済みデータを直に反映。 |

### 4.2 戦略的総括
v11は「壊れないモデル」としての安定性は証明したが、「実務的な提案」を行うには慎重さが勝りすぎている。v12では、この安定した構造を維持したまま、学習パラメータを「攻め」の設定に戻すことで、ガードレールと実用性の黄金比を模索する。
