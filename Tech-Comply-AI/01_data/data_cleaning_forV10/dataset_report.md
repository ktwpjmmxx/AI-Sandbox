# IT法務特化型LLM開発記録：v10 データセット戦略レポート

## 1. 前バージョン（v9）における課題と教訓
前回のファインチューニング（v9）では、特定の出力フォーマットの固定には成功したものの、以下の3つの深刻な課題が発生した。

1. **条文番号のハルシネーション**: 
   7Bクラスのモデルサイズでは膨大な条文番号を正確に暗記できず、存在しない番号を捏造する傾向が顕著であった。
2. **過学習による攻撃的拒絶**: 
   法務データに特化しすぎた結果、専門外の質問（プログラミングや日常会話）に対しても「法律違反である」と強引に結びつけて攻撃的に拒絶する「破滅的忘却」が発生した。
3. **推論能力の低下**: 
   法務コンテキストに脳が占領され、LLM本来の柔軟な論理思考や丁寧な対話能力が損なわれた。

---

## 2. v10における改善戦略（3本の柱）

v9の失敗を糧に、モデルの「理性」と「専門性」を両立させるため、以下の3つの作戦を実行した。

### 【作戦A】情報の抽象化（概念化学習）
- **手法**: 学習データから「第〇条」「第〇項」「第〇号」といった具体的な数字を100%排除。
- **目的**: モデルに「数字の暗記」を強いるのをやめ、「著作権法（翻案権）」や「民法（不法行為責任）」といった**法的ロジック（概念）**の学習に集中させる。
- **効果**: 根拠法令の捏造（ハルシネーション）を物理的に抑制。

### 【作戦B】データの希釈（汎用知能の維持）
- **手法**: 一般的な対話、要約、メール作成、論理パズルなどのデータを約100〜150件混入。
- **目的**: 法務以外のタスクを並行して学習させることで、モデルが元々持っている「良識」と「汎用的な推論能力」を保護する。
- **効果**: 「攻撃的な法律マシーン」化を防ぎ、自然な対話能力を維持。

### 【作戦C】理性的拒絶（ガードレールの構築）
- **手法**: 技術実装やデザイン等の専門外の依頼に対し、法務コンサルタントとして適切に回答を拒絶するデータを約50件追加。
- **目的**: 専門外の質問を無理やり法律に結びつけず、論理的に「専門外であること」を説明し、代替案（エンジニアへの相談など）を提示する挙動を学習させる。
- **効果**: 過学習による暴走を抑制し、実務で使える「誠実な専門家」の振る舞いを実現。

## 3. データクリーニング・プロセス

高品質な学習を実現するため、以下の技術的な整形・洗浄プロセスを実装した。

1. **正規表現による徹底洗浄**: 
   `re.sub(r"第?\d+[条項号](の\d+)?", "", text)` 等の正規表現を用い、データ内に残存していた微細な条文番号や、テンプレートの残骸（「該当条項」タグ等）を一掃。
2. **アップサンプリングによる比率調整**: 
   少数派である「理性的拒絶」データを3倍に複製（アップサンプリング）し、ガードレールとしての機能を強化。
3. **論理的シャッフル**: 
   法務、一般、拒絶のデータが偏らないよう、シード値を固定した上で完全にランダム化。1つのエポック内で多角的な視点を同時に学習できる構造を構築。
4. **フォーマットの最適化**: 
   JSONL内の出力テキストに適切な改行コード（`\n`）を挿入し、視認性の高い「4項目回答フォーマット（リスク、法令、理由、案）」を安定して出力できるよう調整。

## 4. 最終データセット統計
- **総データ件数**: 約650件（重複・複製含む）
- **構成比率**: 
    - IT法務（抽象化済み）: 約72%
    - 一般推論（理性維持）: 約15%
    - 理性的拒絶（ガードレール）: 約13%

## 5. 今後の評価指標
本データセットを用いた学習（v10）では、以下の状態を「成功」と定義する。
- 法的リスクを「概念名」で正しく指摘できること。
- 専門外の質問に対し、怒らず、嘘をつかず、適切に専門外であることを伝えられること。
- 出力フォーマットが崩れず、適切な改行を含んで表示されること。
