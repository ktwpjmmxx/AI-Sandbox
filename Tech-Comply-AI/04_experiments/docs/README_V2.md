## 📈 v2の実測結果

### ✅ 改善された点

1. **データ量の増加**
   - 35件 → 150件達成 ✅
   - 多様なケースをカバー ✅

2. **学習の深化**
   - エポック数5で十分な学習 ✅
   - 評価機能により最良モデルを保存 ✅

3. **モデルの表現力向上**
   - LoRAランク32で細かい調整が可能 ✅
   - 7つのモジュールで広範囲を学習 ✅

### ⚠️ 残った課題

#### 課題1: JSON後に余計なテキスト

**実際の出力例**:
```json
{"リスクレベル": "高", "該当法": "景品表示法", ...}

この広告は消費者庁のガイドラインによれば...（← 余計な説明が続く）
```

**影響**: JSON形式成功率が目標85%に届かず（実測60%程度）

#### 課題2: 出力フォーマットの不統一

- 「リスクレベル」と「【リスクレベル】」が混在
- JSON形式とプレーンテキストが混在
- 一貫性のない出力

#### 課題3: 回答完遂率の伸び悩み

- 目標: 80%
- 実測: 60%程度
- 長文で途切れる問題が残存

---

## 🔍 課題の原因分析

### 原因1: システムメッセージの曖昧さ

**v2のシステムメッセージ**:
```python
"回答は必ず指定されたJSON形式で、それ以外の文章は含めないでください。"
```

**問題点**: 「必ず」という指示だけでは不十分。より強い制約が必要。

### 原因2: 推論パラメータの不適切さ

**v2の推論設定**:
```python
do_sample=False          # Greedy Search
repetition_penalty=1.1   # 控えめな設定
```

**問題点**: JSON後のテキスト生成を止められない

### 原因3: データフォーマットの混在

学習データ内で以下が混在:
- JSON形式のみ
- プレーンテキスト形式
- JSON + 説明文

**問題点**: モデルがどの形式で出力すべきか混乱

---

## 💡 v3に向けた改善提案

### 提案1: データの厳密化

**悪い例（v2）**:
```json
{"output": "{\"リスクレベル\": \"高\", ...} この広告は..."}
```

**良い例（v3）**:
```json
{"output": "{\"リスクレベル\":\"高\",\"該当法\":\"...\",\"理由\":\"...\",\"修正案\":\"...\"}"}
```

→ JSON以外のテキストを完全削除

### 提案2: プロンプトの強化

**v3のシステムメッセージ（案）**:
```python
"回答は必ずJSON形式のみで出力し、JSON以外の文章は一切含めないでください。"
"JSONの開始{から終了}まで以外は何も出力しないでください。"
```

→ より厳格な指示

### 提案3: 推論パラメータの調整

**v3の推論設定（案）**:
```python
temperature=0.3           # より決定的に
repetition_penalty=1.15   # 繰り返し抑制強化
no_repeat_ngram_size=3    # 新規追加
```

## 🎓 v2から学んだこと

### ✅ 成功した点

1. **データ量の重要性**
   - 35件 → 150件で明らかに性能向上
   - 多様性が品質に直結することを確認

2. **評価機能の有用性**
   - 学習中に最良モデルを選択できた
   - 過学習を早期発見できた

3. **LoRA設定の効果**
   - ランク32で表現力が明確に向上
   - より広範囲のモジュールを調整する価値を確認

### ⚠️ 改善が必要な点

1. **データ品質 > データ量**
   - 150件あっても品質が低いと効果薄
   - フォーマット統一が最重要課題

2. **システムメッセージの重要性**
   - 曖昧な指示では不十分
   - より具体的で厳格な指示が必要

3. **推論パラメータの重要性**
   - 学習だけでなく推論設定も重要
   - JSON特化なら決定的な生成が有効
