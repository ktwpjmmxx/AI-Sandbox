「過学習（Overfitting）」と「学習率の過剰」による「破滅的忘却（Catastrophic Forgetting）」の複合要因

ログにある train_loss: 0.24 という数値は、たった135件のデータセットに対しては低すぎる。モデルは「法律の論理」を学習したのではなく、「学習データのテキストの並び」を丸暗記しようとして、元の言語モデルが持っていた論理能力（日本語のつながりや文脈理解）を破壊してしまった。

【原因の特定】

1. 学習率が高すぎる（最大の原因）

　- 現状: learning_rate: 2e-4 (0.0002)

　- 分析: 7Bパラメータのモデルに対し、わずか150件程度のデータでこの学習率は高すぎる。重みが急激に更新されすぎたため、モデルが元々持っていた知識が壊れ、法律用語をデタラメに並べる状態（ハルシネーション）になっている。

　- 症状: 推論結果で「民法502条（信義則）」や「消費者契約法第13条」など、文脈と無関係な条文や用語が羅列されている。

2. 学習データの丸暗記とEOSトークンの無視

現状: 推論結果の末尾に 入力: 新作ゲームの... と続いてしまっています。

分析: モデルが「回答が終わったら止まる（EOSトークンを出す）」ことを学習できず、「回答の次には、次の学習データの『入力:』が続く」というデータの並び順自体を学習してしまっています。

症状: 生成が止まらず、学習データ内の別のデータを出力し始める。

3. データ量に対するEpoch数のバランス

現状: 5 Epoch
分析: データが少ない場合、Epochを重ねるとすぐに過学習します。このデータ量なら、学習率を下げて丁寧に回す必要があります。

手順1: config_v4.yaml の修正（最優先）

学習率を1/4に下げ、正則化（weight_decay）を強め、LoRAのドロップアウトを増やして過学習を防ぎます。

手順3: inference_v4.py の修正（生成停止条件の強化）
推論時に、モデルが暴走して「次の入力」まで喋り出さないように、強制的に止めるキーワードを設定します。

inference_v4.py の generate_response メソッドを以下のように書き換えてください。

def generate_response(self, instruction, input_text):
        prompt = (
            f"<s>[INST] <<SYS>>\n{self.system_message}\n<</SYS>>\n\n"
            f"{instruction}\n\n入力:\n{input_text} [/INST]"
        )
        
        inputs = self.tokenizer(prompt, return_tensors="pt").to(self.model.device)
        
        # 暴走防止: '入力:' という単語が出たらそこで打ち切る設定等は
        # TransformersのStoppingCriteriaを使うのが正式ですが、
        # 簡易的にはEOSトークンを信じるしかありません。
        # ここではパラメータを少し保守的にします。

        with torch.no_grad():
            outputs = self.model.generate(
                **inputs,
                max_new_tokens=512,
                do_sample=True,
                temperature=0.3,       # 0.7 → 0.3 (ハルシネーション抑制のため低くする)
                top_p=0.9,
                repetition_penalty=1.2, # 繰り返しペナルティを少し強く
                eos_token_id=self.tokenizer.eos_token_id,
                pad_token_id=self.tokenizer.eos_token_id,
            )
        
        response = self.tokenizer.decode(outputs[0], skip_special_tokens=True)
        
        if "[/INST]" in response:
            response = response.split("[/INST]")[-1].strip()
        
        # 【追加対策】もしモデルが次の「入力:」まで生成していたら、そこで切り落とす
        if "入力:" in response:
            response = response.split("入力:")[0].strip()
            
        return response

学習率 5e-5: 既存の日本語能力（文法や論理）を壊さずに、新しい知識（コンプライアンスのフォーマット）だけをそっと上書きします。

temperature=0.3: 推論時の「創造性」を抑え、最も確実性の高い（学習データに近い）回答を選ばせることで、デタラメな法律用語の生成を防ぎます。

事後処理: 万が一学習データが漏れ出しても、Python側で split("入力:") することで、ユーザーには綺麗な回答だけを見せることができます。

次のアクション
config_v4.yaml の学習率を 5e-5 に書き換えてください。

再度 python train_v4.py を実行してください（モデルの保存先が変わるよう output_dir を変更するのを忘れずに）。

学習完了後、inference_v4.py の生成パラメータ（temperature等）を調整した上で実行してください。
